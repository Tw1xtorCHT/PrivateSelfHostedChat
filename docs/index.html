PrivateSelfHostedChat

Private self-hosted chat with P2P voice calls. No registration, no logs, no traces. Deploy on any VPS in 3 minutes with a single command.

Custom engine - Node.js + Socket.io + WebRTC

💬 Anonymous Chat

No registration or accounts. Enter a link, type a password, chat. Close the tab and everything disappears.

📞 P2P Voice Calls

Voice calls via WebRTC with audio level indicators. Traffic goes directly between browsers. Server never hears the conversation.

🔒 Auto-rotating Password

Password changes automatically every hour. New password sent to Telegram. Old sessions destroyed. Users auto-kicked.

🤖 Telegram Bot

Manage chat via bot. One tap to reset chat, change password, get notified.

🗃 File Sharing

Drag and drop files up to 50 MB. PDFs, photos, documents. Stored in RAM only, wiped on reset.

🔓 HTTPS

Automatic SSL certificate via Let's Encrypt. All traffic encrypted. Zero config.

📷 Zero Disk Storage

Messages and files live in RAM only. Nothing is written to disk. Container restart equals full wipe.

⚙ 4 Containers

Minimal footprint: chat engine, nginx, caddy, coturn. Runs on 1 GB RAM VPS.

Install in 3 minutes

Step 1 - Create a Telegram bot

Open @BotFather, send /newbot and copy the token.
Get your Chat ID from @userinfobot.

Step 2 - Get a free domain

Go to duckdns.org, create a subdomain and point it to your server IP.

Step 3 - Run the installer

SSH into your VPS and run:

curl -fsSL https://raw.githubusercontent.com/Tw1xtorCHT/PrivateSelfHostedChat/main/install.sh | bash

Architecture

Browser → HTTPS (Caddy) → Nginx → PSChat Engine
Auth: token-based, built into the engine
Calls: WebRTC P2P, direct between browsers
TURN: Coturn fallback if P2P fails
Management: Telegram Bot → auto-reset every hour

Security

Zero disk storage Messages and files exist only in RAM. Container restart wipes everything permanently.
Open source Full source code on GitHub. No backdoors, no analytics, no trackers.
Your server Data stays on your VPS. No third parties, no clouds, no corporations.
P2P calls Audio goes directly between devices. Server only connects peers, never hears audio.
